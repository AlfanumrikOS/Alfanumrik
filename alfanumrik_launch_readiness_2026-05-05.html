🔴 Critical Blockers — Must Fix Before Any Launch

C-01 · CRITICAL Quiz Score Data Type Mismatch — XP Points Stored, Not Percentage
The quiz_sessions.score column stores XP points (10 per correct answer), not percentage. A perfect 20-question quiz has score = 200. Recent data confirms: score=200 with 20 questions (1000%), score=10 with 5 questions (200%). The average stored score is 113.3 — clearly not a percentage.

Risk: Any UI component that renders score directly as a percentage will show "1000%" to a student who scored perfectly. This is a student-facing embarrassment and a credibility killer. The correct formula is: percentage = (score / (total_questions × 10)) × 100.
Fix Required 1. Grep all UI files rendering quiz score for direct use of score field without this formula.   2. Confirm all SubjectProgress, quiz result pages, and parent dashboard use the correct calculation.   3. Add a DB-level computed column or a typed helper function get_score_pct() to prevent future drift.
C-02 · CRITICAL Pricing Mismatch — DB Plan "Unlimited" at ₹1,499 vs Investor Deck "Family/School" at ₹1,099
The subscription_plans table has: Explorer ₹0 · Starter ₹299 · Pro ₹699 · Unlimited ₹1,499/month. The official corporate identity document (used in investor materials, pre-seed deck) states: Starter ₹299 · Pro ₹699 · Family/School ₹1,099/month. There is a ₹400/month discrepancy and a naming mismatch on the top tier.

Risk: If an investor checks the live pricing page and sees ₹1,499, but the deck says ₹1,099, it raises questions about data integrity and governance. If a real parent pays ₹1,499 when told ₹1,099, it is a compliance issue.
Fix Required Decide canonical price before launch and sync: either (a) update DB plan to ₹1,099 + rename "Unlimited" → "Family / School" + update Razorpay plan plan_Sb0bD6umoH12gf, or (b) update investor deck to ₹1,499. One source of truth. Both must match.
C-03 · CRITICAL 36 Orphan Edge Functions in Production — Untracked, Unreviewed
Production has 71 Edge Functions deployed but only 35 exist in the source repo. The drift was documented in PR #544 but the orphans were NOT decommissioned — they remain live and callable. These 36 functions cannot be code-reviewed, cannot be tested, and their logic is unknown. Any production API call that accidentally hits one of these (via shared naming, edge case routing, or documentation references) returns undefined behaviour.

Risk: Unreviewed logic running in production. Security exposure. Cannot be audited. Blocks SOC2/ISO compliance path.
Fix Required Pull the list of 36 orphan function slugs from the drift report. For each: (1) verify it is not called by any live client or Edge Function. (2) Deploy a tombstone (HTTP 410 Gone) or delete via Supabase Dashboard. (3) Commit a "production-function-inventory.json" to repo so this never drifts again.
