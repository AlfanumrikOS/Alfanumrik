# Pedagogy v2 — Wave 2 (Weekly Curiosity Dive) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Weekly layer of the Three-Speed Learning Rhythm — a 60-minute self-directed curiosity dive that runs once per ISO week, persona-adaptive, producing one student-edited artifact saved to the lab notebook.

**Architecture:** A new `/dive` route hosts a three-option picker (suggested phenomenon, weak-topic dive, own topic). Picking one opens a new Foxy "explorer" mode — Socratic-led but allowed to give direct exposition when stuck, RAG-grounded, structurally producing a draft artifact as the conversation progresses. The artifact lands in the existing `/lab-notebook` surface tagged as the week's piece. A weekly streak token (forgiving — missing one week is fine, four consecutive misses resets) persists alongside existing daily-streak storage.

**Tech Stack:** Next.js 16 App Router, React 18, Vitest, Playwright, Supabase (Postgres + RLS), Claude Haiku via existing Foxy AI workflows, RAG via existing match_rag_chunks_ncert_rrf RPC. Path alias `@/*` → `./src/*`.

**Spec:** [docs/superpowers/specs/2026-05-08-pedagogy-v2-three-speed-rhythm-design.md](../specs/2026-05-08-pedagogy-v2-three-speed-rhythm-design.md) §5.2 and §11.
