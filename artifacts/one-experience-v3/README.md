# One Experience V3 review evidence

This directory is populated from the production build during release review.
Evidence is valid only when its recorded commit matches the pull request head.

Required output:

- phone, tablet and desktop screenshots for all five role homes;
- the 10-viewport responsive matrix results;
- populated, loading, empty, stale, error and denied states;
- production preview-route 404 evidence;
- accessibility, keyboard and reduced-motion results;
- build ID, route count, bundle budget and test summaries;
- rollout preflight and rollback verification.

Historical screenshots must not be used to certify a rebuilt source tree.
